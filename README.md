## 什么人适合用 SmartSVN？

先说一个挺常见的场景：你在公司用 Windows 台式机，家里是 MacBook，偶尔还会 SSH 进 Linux 服务器跑点东西。如果这三个地方都要碰 SVN，你要么装三套不同的客户端，要么每次都开终端敲命令行。

SmartSVN 的解法很简单——三个平台装一样的软件，界面、操作逻辑、快捷键全都一致，换了台机子不用重新适应。

适合用的人大概是这几类：

- 长期维护 SVN 仓库的后端/全栈开发者
- 需要管理分支和标签、频繁做代码合并的团队
- 同时使用多个操作系统的开发者
- 想把 SVN 操作可视化、减少命令行依赖的技术人员

第一次打开 SmartSVN，项目目录视图里能直接看到文件状态：**红色箭头**表示本地有修改，**绿色箭头**表示仓库里有更新——不用去猜哪些文件需要提交、哪些需要拉取，一眼全看到。
